Online Algorithms: Recurrence formulas for mean and variance

Goal

Derive simple online (recurrence) formulas for the arithmetic mean and variance, and show a tiny example implementation. The presentation is kept as simple as possible.

1) Incremental (online) mean — derivation

Let x_1, x_2, ..., x_n be observations. The arithmetic mean after n observations is

M_n = (1/n) * \sum_{i=1}^n x_i

We can express M_n from M_{n-1} and the new value x_n:

M_{n-1} = (1/(n-1)) * \sum_{i=1}^{n-1} x_i

Multiply M_{n-1} by (n-1) and add x_n:

(n-1)M_{n-1} + x_n = \sum_{i=1}^{n} x_i

Divide by n:

M_n = \frac{(n-1)M_{n-1} + x_n}{n}

Rearranged into a common, numerically stable update form:

M_n = M_{n-1} + \frac{x_n - M_{n-1}}{n}

Intuition: adjust the previous mean by a fraction (1/n) of the difference between the new value and the previous mean.

2) Incremental variance — derivation (Welford's method)

A stable online algorithm keeps a running mean M_n and a running sum-of-squares-of-differences, called M2 (not to confuse with mean).

For the nth value x_n, define:

delta = x_n - M_{n-1}
M_n = M_{n-1} + delta / n
M2_n = M2_{n-1} + delta * (x_n - M_n)

From M2 you get variance estimates:

  • Population variance: sigma^2 = M2_n / n
  • Sample variance (unbiased): s^2 = M2_n / (n - 1)  (for n > 1)

Why online algorithms matter

Online (one-pass) algorithms update statistics incrementally as each new observation arrives. They are essential when data arrive as a stream, when the dataset is too large to fit into memory, or when you need low-latency updates. Online updates require O(1) memory (constant extra space) and O(1) time per observation, making them efficient for real-time analytics.

Numerical stability is a critical concern: naive formulas that compute variance as E[X^2] - (E[X])^2 can suffer catastrophic cancellation when sums are large and differences are small. Welford's method avoids this by keeping a running centered sum (M2), which prevents loss of precision and produces accurate results even for long streams.

Use cases: telemetry and monitoring, finance (streaming returns), online learning, sensor networks, and any situation where summaries must be computed on-the-fly without storing the full dataset.

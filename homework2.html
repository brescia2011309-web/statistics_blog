Explain the concept of dataset and distribution

DATASET

A dataset is a structured collection of observations, usually organized in rows (cases) and columns (variables).

For example, a dataset might record characteristics of individuals, focusing on categorical variables: 

Person_ID
Eye_Color (e.g., 'Blue', 'Brown', 'Green')
Hair_Color (e.g., 'Blonde', 'Dark', 'Red')

Each row is one observation; together they form the dataset.

DISTRIBUTION

A distribution, on the other hand, describes how the values of a variable are spread across possible outcomes.

For categorical data, the distribution is defined by the count of individuals within each possible category (e.g., 'Blue' eyes vs 'Brown' eyes).

The key distinction is simple:

The dataset provides the initial observations that you've collected. The distribution reveals the underlying pattern of variability within that data, allowing you to understand how your observations truly behave.

Calculating the univariate distribution means summarizing the collected observations of a single variable to understand its shape, central tendency, and variability. Since the variables are categorical, the distribution is expressed through frequency tables (Absolute Frequency).

SQL query that would produce the same result:

Univariate Distribution (Absolute Frequencies) Example: 

SELECT
    Eye_Color,
    COUNT(*) AS Absolute_Frequency
FROM
    People
GROUP BY
    Eye_Color;

This single query can be reused to calculate the distribution of any categorical variable simply by replacing the column name (for example, replacing Eye_Color with Hair_Color) in the SELECT and the GROUP BY clauses.
